Markdown Ready Support

Markdown Ready is a macOS menu-bar app for converting PDFs, scanned pages, and document images into Markdown and optional JSON files on your Mac.

Quick Start

  1. Open Markdown Ready from the menu bar.
  2. Select a PDF, JPG/JPEG, PNG, or input folder.
  3. Choose an output folder.
  4. Choose Markdown and/or JSON output.
  5. Use table options if your document contains important tables.
  6. Start conversion.
  7. Open Last Result or the selected output folder to review the generated file.

Supported Inputs

  • PDF
  • JPG/JPEG
  • PNG
  • Folders containing supported document files

Supported Outputs

  • Markdown (.md)
  • Optional JSON sidecar (.json)

Folder Watching

Folder watching is designed for hands-off batches. Choose an input folder and an output folder, then start watching. While the app is running, supported files placed in the input folder are converted into the output folder.

  • The app stores local fingerprints so repeated scans can skip files already processed by the watched workflow.
  • If you edit or replace a source file and want to process it again, use the app's clear/reset workflow where available.
  • Keep the app running while using folder watching.

Performance And Power

Markdown Ready runs local OCR on your Mac. Large PDFs, image-heavy documents, and folder batches can use sustained CPU/GPU and unified memory. For long conversions, connect your Mac to power and avoid closing the lid.

Accuracy And Review

OCR output can contain mistakes. Complex tables, equations, handwriting, poor scans, screenshots, rotated pages, and tiny text need review. Markdown Ready prepares documents for reuse; it does not verify facts or replace human judgment.

Privacy

The app processes selected documents locally and does not upload document contents to a cloud OCR API. Settings, selected folder access bookmarks, and processed-file fingerprints are stored locally on your Mac. Review the Markdown Ready Privacy Policy for details.

Troubleshooting

The app cannot access a folder after relaunch

Choose the input/output folder again from the app so macOS can refresh the user-selected folder permission.

Conversion is slow

Large documents can take time because OCR runs locally. Try a smaller test file first, keep the Mac connected to power, and avoid running other heavy workloads at the same time.

Output is missing a table or formula

Enable the table-related options, convert again, and review the output. Very complex tables and formulas may still need manual correction.

I do not see a new file in the output folder

Confirm that an output folder is selected, the source file type is supported, and the app has permission to access both folders.
